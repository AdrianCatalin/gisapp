 94 /**
 95  * Replace layer name in GetFeatureInfo request (identify and attributes table) and in layer export. Useful to create database view for layer, where you can reorder fields or create new read only fields from geometry, like: area, length, X, Y. This data will be displayed in identify window and attributes table and used in layer export instead of data from original layer.
 96  * You could use view in QGIS in the first place for ordinary layers, but if you wish that layer is editable it cannot be connected to view.
 97  * > Prepare:
 98  * 1. Write in this array layer name as it is in QGIS project,
 99  * 2. Create corresponding view named as *your_layer_name_view* and add it to the QGIS project. View will be hidden from legend and map.
100  * 3. Set view symbology to transparent or no symbol, to avoid "seeing" it in print, because it must be added to print layers in case if its table is used in print layout.
101  *
102  * > If layer name is **"points"**, view name must be **"points_view"**, add in array only **"points"**
103  * @example "['points']";
104  * @type {Array}
105  */
